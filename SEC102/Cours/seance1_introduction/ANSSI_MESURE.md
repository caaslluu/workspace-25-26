# 5) Gestion des comptes privilégiés

Tenir à jour l’inventaire des comptes privilégiés :  

- Liste actualisée des comptes avec leurs droits dans le SI  
- Listing des comptes admin Windows/Linux, compte root, comptes de service, bases de données, etc.  
- Documentation sur les comptes : qui utilise le compte ? Sur quels systèmes ? (exploit, prod, etc.)  
- Lister les comptes créés et supprimés

# 34) Politique de mise à jour des composants du SI (OS, logiciels, etc.)

- **Inventorier les composants du SI** → OS, logiciels, applications, équipements.  
- **Définir une fréquence de mise à jour** → correctifs critiques, sécurité, mises à jour majeures.  
- **Tester les mises à jour avant déploiement** → environnement de test ou pré-production.  
- **Planifier le déploiement** → fenêtres de maintenance, communication aux équipes.  
- **Automatiser quand possible** → WSUS, scripts, outils de patch management.  
- **Contrôler et tracer les mises à jour** → journalisation, rapports de conformité.  
- **Gérer les comptes privilégiés** → inventaire, documentation, création/suppression suivies.  
- **Réviser régulièrement la politique** → adaptation aux évolutions du SI et des menaces.
